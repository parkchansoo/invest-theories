# Filing Guide — 한국·미국 공시 체계 활용 스킬

이 스킬은 투자 분석에 필요한 공시를 **정확히 찾고, 올바르게 해석하고, 투자 판단에 연결**하는 것을 돕는다.

## 핵심 원칙

1. **"알고 싶은 것 → 확인할 공시"를 즉시 연결한다**: 사용자가 알고 싶은 정보가 어떤 공시의 어떤 항목에 있는지 바로 안내한다.
2. **한국과 미국의 차이를 이해한다**: 같은 이벤트도 한국과 미국에서 다른 공시 체계로 보고된다.
3. **투자 영향도를 기준으로 우선순위를 매긴다**: 모든 공시가 같은 중요도가 아니다. CRITICAL/HIGH 공시를 먼저 확인한다.

## 투자자 Quick Reference — 무엇을 알고 싶을 때 어디를 볼 것인가?

### 한국 종목

| 알고 싶은 것 | 확인할 공시 | DART 코드 | 스크립트 |
|---|---|---|---|
| 연간 사업 내용, 재무, 리스크 | **사업보고서** | A001 | `fetch_dart.py --type 사업보고서` |
| 분기 실적 | **분기보고서** | A003 | `fetch_dart.py --type 분기보고서` |
| 잠정실적 (어닝 서프라이즈) | **공정공시 — 잠정실적** | I002 | KIND 확인 |
| 수주, 배당결정, 대표이사변경 | **거래소 수시공시** | I001 | KIND 확인 |
| 증자, CB/BW 발행 | **주요사항보고서** | B001 | `fetch_dart.py --type 주요사항보고서` |
| 내부자(임원·주요주주) 거래 | **지분공시** | D002 | DS004 API |
| 5% 이상 대량보유 변동 | **대량보유상황보고서** | D001 | DS004 API |
| 경영진 보수, 지배구조 | **사업보고서** DS002 | A001 | DS002 API |
| 감사의견 (적정/한정/부적정) | **감사보고서** | F001 | DS002 API 20번 |
| 계열사 내부거래 (대기업) | **공정위 대규모내부거래** | J001 | DART 목록 조회 |
| 기업집단 지배구조 전체 그림 | **기업집단현황공시** | J004 | egroup.go.kr |

## 투자 영향도별 공시 분류

### CRITICAL — 즉시 확인 필수

**한국**: 부도발생, 영업정지, 회생절차, 해산사유, 감사의견 비적정, 상장폐지 관련
**미국**: 8-K 1.03(파산), 4.02(재무제표 신뢰불가), 5.01(경영권변경), 3.01(상장폐지), 2.02(실적발표)

### HIGH — 주가 직접 영향

**한국**: 유상증자, CB/BW 발행, 합병/분할, 대량보유변동, 자기주식 취득/처분, 수주공시, 잠정실적
**미국**: 8-K 1.01/1.02(계약), 2.01(M&A완료), 2.03(채무발생), 2.06(손상차손), 5.02(임원변동), 7.01(Reg FD)

### MEDIUM — 중기 분석용

**한국**: 소송, 영업양수도, 유형자산 양수도, 사외이사변동, 공정위 내부거래
**미국**: 8-K 2.05(구조조정), 3.02(미등록주식), 5.03(정관변경), 5.07(주총결과)

## 한국 vs 미국 공시 비교 — 핵심 차이

| 구분 | 한국 | 미국 |
|---|---|---|
| 규제기관 | 금감원 + 거래소 + 공정위 (3중) | SEC (단일) |
| 전자공시 | DART + KIND + egroup | EDGAR |
| 공시 시한 | **당일~1일** (더 빠름) | 4영업일 |
| 공시 철학 | 열거주의 (규칙 기반) | 원칙주의 (중요성 기반) |
| 중요성 기준 | 정량적 (자산의 10%, 매출의 5%) | 판례 기반 ("합리적 투자자") |
| 구조화 데이터 | OpenDART API (83개) | EDGAR API + XBRL |
| 사이버보안 공시 | **없음** | 8-K 1.05 (2023년 신설) |
| Reg FD 통합 | 별도 체계 (자율공시) | 8-K 7.01로 통합 |

### 미국에만 있는 주요 공시
- **8-K 1.05**: 사이버보안 사건 (한국에 해당 의무 없음)
- **8-K 4.02**: 재무제표 신뢰불가 선언 (한국은 감사보고서 정정으로 처리)
- **8-K 2.06**: 자산 손상차손 (한국은 별도 항목 없음)

### 한국에만 있는 주요 공시
- **유상증자 결정** 독립 공시 (미국은 S-3/8-K로 분산)
- **CB/BW 발행 결정** 독립 공시
- **공정위 대규모내부거래** (재벌 감시 목적, 미국에 없음)
- **비상장 계열사 중요사항** (미국에 없음)

## 공시 분석 시 주의사항

### 정정공시 처리
- 한국: `report_nm`에 `[기재정정]`, `[첨부정정]` 접두사 → 최신 정정본이 유효
- 미국: Form 뒤에 `/A`가 붙으면 정정 (예: 10-K/A, 8-K/A)
- **정정공시 자체가 투자 시그널**: 무엇이 바뀌었는지가 중요

### 8-K 아이템 코드 해석
하나의 8-K에 여러 아이템이 동시에 보고되는 경우가 많다. `items` 필드가 "2.02,9.01"이면 실적발표(2.02) + 첨부서류(9.01).

### 한국 거래소 공시(I001, I002) 특이점
- 구조화 API 없음 → HTML 파싱 필요
- 같은 유형이라도 회사마다 양식이 다름
- 코스피 vs 코스닥 양식 차이 존재
